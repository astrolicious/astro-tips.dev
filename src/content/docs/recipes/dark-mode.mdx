Having light and dark theme is almost neccessary for a website, and there are many ways to handle this. This recipe will cover one method which follows best practices and keeps DX in mind.

This recipe will be for two components, one that managers the themes and creates the API, and one that the client interacts with to manually select the theme of their choosing. We will also go over styling and usage of the components.

Here's a [StackBlitz](https://stackblitz.com/edit/withastro-astro-g1fwa8) example of this recipe, and here is an [example repo](https://github.com/astrolicious/astro-tips.dev/tree/main/examples/dark-mode).

## Key Features

- Compatible with [View Transitions](https://docs.astro.build/en/guides/view-transitions/) and uses [Web Components](https://developer.mozilla.org/en-US/docs/Web/API/Web_components)
- Includes an optional [`<select>`](#theme-select-component) component in this recipe and a [`<button>`](https://github.com/astrolicious/astro-tips.dev/tree/main/examples/darkmode/src/components/ThemeToggle.astro) in the [example](https://github.com/astrolicious/astro-tips.dev/tree/main/examples/darkmode) on GitHub
- Dispatches a custom `theme-changed` event that can be listened for and gives access to `event.details.theme` and `event.details.systemTheme`
- Allows for easily setting the default theme with a prop to the component
- Exposes `theme` global on client for a nice API:
  - `theme.getTheme()`
  - `theme.setTheme()`
  - `theme.getSystemTheme()`
  - `theme.getDefaultTheme()`

## Theme Manager Component

This component will live in the `<head>` of your layout or pages, it is responsible for ensuring there is no [FOUC](https://en.wikipedia.org/wiki/Flash_of_unstyled_content) and creating the developer facing `theme` client side API.

<Steps>

1. This component starts off with an inline script that handles the themes by creating a `window.theme` api. We will put the window.theme definition inside an [IIFE](https://developer.mozilla.org/en-US/docs/Glossary/IIFE), so that we can avoid polluting the window and have control over what exactly is made global.  

    ```astro
    <theme-manager data-default-theme={defaultTheme}></theme-manager>
    <script is:inline>
      if (!customElements.get("theme-manager")) {
        class ThemeManager extends HTMLElement {
          connectedCallback() {
            this.initThemeManagement(this.dataset.defaultTheme || "auto");
          }

          initThemeManagement(defaultTheme) {
            window.theme = (() => {
              // ...
            })();

            theme.setTheme(theme.getTheme());
          }
        }
        customElements.define("theme-manager", ThemeManager);
      }
    </script>
    ```
2. Now we can get into the first part of `window.theme`, creating our store. We need to check if `localStorage` is available to us, because it isn't available everywhere, and not break when it isn't.
    
    ```js
    window.theme = (() => {
      const storageKey = "theme";
      const store =
        typeof localStorage !== "undefined"
          ? localStorage
          : { getItem: () => null, setItem: () => {} };

      // ...

    })();
    ```
3. Next let's listen for device setting changes, so when in `auto` mode the theme will respond to clients changing their device settings, and in order to do that we need to also create the `applyTheme` function. 
    ```js
    window.theme = (() => {

      // ...

      const mediaMatcher = window.matchMedia("(prefers-color-scheme: light)");
      let systemTheme = mediaMatcher.matches ? "light" : "dark";
      mediaMatcher.addEventListener("change", (event) => {
        systemTheme = event.matches ? "light" : "dark";
        applyTheme(theme.getTheme());
      });

      function applyTheme(theme) {
        document.documentElement.dataset.theme =
          theme === "auto" ? systemTheme : theme;
        document.dispatchEvent(
          new CustomEvent("theme-changed", {
            detail: { theme, systemTheme },
          })
        );
      }

      // ...

    })();
    ```
4. Now finally let's create the functions that will become our developer facing API, which designed to give optimal DX when working with this theme provider. Any function we return here will be made available client side on the global `theme`, like `theme.anything()`
    ```js
    window.theme = (() => {

      // ...

      function setTheme(theme = defaultTheme) {
        store.setItem(storageKey, theme);
        applyTheme(theme);
      }

      function getTheme() {
        return store.getItem(storageKey) || defaultTheme;
      }

      function getSystemTheme() {
        return systemTheme;
      }

      function getDefaultTheme() {
        return defaultTheme;
      }

      return { setTheme, getTheme, getSystemTheme, getDefaultTheme };
    })();
    ```

</Steps>

## Theme Select Component

Of course we need a way to allow users to switch between themes, and for this recipe we will go over a basic `<select>` based element. A more complex theme toggle button is included in the [examples](https://github.com/astrolicious/astro-tips.dev/tree/main/examples/darkmode) on this repository's GitHub and on the [StackBlitz](https://stackblitz.com/edit/withastro-astro-g1fwa8) included with this recipe.

<Steps>

1. Get started with another inline script that is defining a [custom element](https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_custom_elements)

    ```astro
    <theme-selector></theme-selector>
    <script is:inline>
      if (!customElements.get("theme-selector")) {
        customElements.define(
          "theme-selector",
          class extends HTMLElement {
            // ...
          }
        );
      }
    </script>
    ```
2. Next set up the `connectedCallback` and methods of our component, with the goal of basically just creating a `<select>` component that sets the options correctly based on the current `theme` and to listen for the `theme-changed` event and respond accordingly.

    ```js
    class extends HTMLElement {
      connectedCallback() {
        this.innerHTML = `
          <select>
            <option value="auto">Auto</option>
            <option value="light">Light</option>
            <option value="dark">Dark</option>
          </select>
        `;
        this.querySelector("select").onchange = (event) =>
          theme.setTheme(event.target.value);
        this.setAttribute("aria-label", "Select Theme");
        this.updateSelectedTheme();

        document.addEventListener("theme-changed", (event) => {
          this.updateSelectedTheme(event.detail.theme);
        });
      }

      updateSelectedTheme(newTheme = theme.getTheme()) {
        this.querySelector("select").value = newTheme;
      }
    }
    ```

</Steps>


## Styles

So obviously our theme solution wouldn't be complete without styling the different themes! This can be done many ways of course, but in essence we will be setting up css varibles according to the `data-theme`. Here's a basic example of how this would be done:

```astro
<style is:global>
  :root[data-theme="light"] {
    --background-color: #ffffff;
    --text-color: #000000;
  }

  :root[data-theme="dark"] {
    --background-color: #333333;
    --text-color: #ffffff;
  }

  body {
    background-color: var(--background-color);
    color: var(--text-color);
  }
</style>
```

## ESLint and TypeScript

If you want to use this `window.theme` API inside a normal `<script>` you might want to add it as a property of `Window` in `env.d.ts`.

If you're using ESLint there's a good chance you'll run into `'theme' is not defined` due to the [no-undef](https://eslint.org/docs/latest/rules/no-undef) rule. To solve this we can add `theme` as a global in `eslintrc.cjs`.


<Tabs>
  <TabItem label="TypeScript">
    ```ts title="src/env.d.ts" ins={3-10}
    /// <reference types="astro/client" />

    interface Window {
      theme: {
        setTheme: (theme: "auto" | "dark" | "light") => void;
        getTheme: () => "auto" | "dark" | "light";
        getSystemTheme: () => "light" | "dark";
        getDefaultTheme: () => "auto" | "dark" | "light";
      };
    }
    ```

## Usage

```astro title="Layout.astro" {10,15,19-26}
---
import ThemeProvider from "./ThemeManager.astro";
import ThemeSelect from "./ThemeSelect.astro";
import { ViewTransitions } from "astro:transitions";
---

<html lang="en">
  <head>
    <!-- ... -->
    <ThemeProvider default-theme="auto" />
    <ViewTransitions />
  </head>
  <main>
    <body>
      <ThemeSelect />
      <slot />
    </body>
  </main>
  <style is:global>
    :root[data-theme="light"] {
      /* CSS variables */
    }
    :root[data-theme="dark"] {
      /* CSS variables */
    }
  </style>
</html>
```

```astro
      }

      initThemeManagement(defaultTheme) {
        window.theme = (() => {
          const storageKey = "theme";
          const store =
            typeof localStorage !== "undefined"

              : { getItem: () => null, setItem: () => {} };

          const mediaMatcher = window.matchMedia(
            "(prefers-color-scheme: light)"
          );
          let systemTheme = mediaMatcher.matches ? "light" : "dark";

            systemTheme = event.matches ? "light" : "dark";

            document.documentElement.dataset.theme =
              theme === "auto" ? systemTheme : theme;
            document.dispatchEvent(
              new CustomEvent("theme-changed", {
                detail: { theme, systemTheme },
              })

            store.setItem(storageKey, theme);

            return systemTheme;

            return defaultTheme;

      }
```
